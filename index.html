Understanding RGB

To understand how we get color values from hexidecimal numbers we first need to understand how the RGB color system works. RGB is most commonly found on electronics. It is made of three primary colors, red, greeen, and blue. RGB is considered an additive color gammet. Meaning that when all the colors are added together you see the color white.

If you look closely at a monitor you will notice groupings of the primary colors are positioned next to eachother. Each emmit its own color of light at different intensities. The combination of color and intensity creates the color we precieve on the screen.

Each primary color in the RGB spectrum is represented by a numerical value between 0 & 255. 0 meaning that primary color is emmiting no light, and 255 meaning the color is at full intensity. So the RGB value 0, 0, 0 would equate to black because no light is being emitted from any of the primary colors. 255, 255, 255 would equate to white because all primary colors are emitting their full brightness.

The Hexidecimal Part

So what the does hexadecimal have to do with RGB?? Good question, lets answer that. All hexadecimal colors will have six digits. (well most of them. There is the possibility of eight digits, but it's less common and supported by few.)

#FFFFFF

When compared against RGB we can break down the hexadecimal into pairs. The first pair of digits in hexacimal is our first value for RGB i.e. red. The next pair for green, and last for blue.

#FF(red values)FF(green values)FF(blue values)

But how does FF = 255?? Well without explaining the entire base 16 system (pro-tip... read the first damn link) An "F" in the second digit place means there are 15 16's (15*16). An "F" in the first digits place equates to a strait 15. 

So FF can also be expressed as (15*16)+15. Which of course equals 255

So in hexadecimal FF is the same as 255, which in RGB means the brightest value for that paricluar primary color.

Bonus

Earlier I mentioned there is a possibility for eight digits in the hexadecimal number. This is used by android systems for transparancy. The hexadecimal is still split by pairs but now the first pair is the transparency, then red, green, and last blue. The transparency is actually controlling the alpha chanel. You can learn more about an alpha chanel here.

Transparency is normally measured by percentage. i.e. 15% transparency means 15% of light (or the back ground) can pass through. Its challenging to creat this percent using hexadecimals.

Remembering that screens translate color in RGB we need to base our hexcolors accordingly. So if you wanted a 20% opaque color we would need to find 20% of 255 (because RGB has values between 0-255 remember?!?). Once we have that value its time to grab the hex color.

255 * .20 = 51

Sometimes this step will leave you with a non whole number. In that case you would need to round up or down.

51 in hexadecimal is #33. So #33DDDDDD would be a dark color that is 20% transparent.
